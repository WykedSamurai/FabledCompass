# MVP Scope

## Release Goal

Prove that a user can discover jobs, complete a workplace scenario, receive useful feedback, and display an earned badge on a profile.

## Included

### Accounts
- Sign up
- Log in
- Choose job-seeker or employer account type

### Profiles
- Name
- Headline
- Location
- About section
- Avatar
- Resume link or upload placeholder
- Earned badges

### Job Board
- Browse jobs
- Search by keyword
- Filter by location and work arrangement
- View job details
- Employer can create, edit, and close a listing

### Scenario System
- Scenario library
- One branching scenario: Service Recovery Constellation
- Authored choices
- Transparent scoring
- Results screen
- Structured feedback

### Badge System
- One badge: Customer Service Foundations
- Badge criteria
- Date earned
- Scenario source
- Profile display

### Employer View
- View applicant profile
- View earned badges and their criteria

## Not Included in the MVP

- Social feed
- Guilds or groups
- Direct messaging
- Multiplayer roleplay
- AI-generated scenario dialogue
- Voice interaction
- Paid subscriptions
- Employer-created scenarios
- Automated job imports
- Advanced applicant ranking
- Mobile applications
- Public comments or forums

## Required Pages

1. Landing page
2. Sign-up page
3. Login page
4. Profile setup
5. Job board
6. Job details
7. Scenario library
8. Scenario player
9. Scenario results
10. User profile
11. Employer dashboard
12. Job posting form

## Definition of Done

The MVP is complete when a new user can create an account, complete the first scenario, meet or fail its badge criteria, view clear feedback, and see an earned badge on their profile. An employer account must be able to post a job and view a candidate profile.

## Scope Rule

Any feature that does not directly support the definition of done belongs in a later release.
