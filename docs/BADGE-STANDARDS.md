# Badge Standards

## Purpose

Badges represent demonstrated performance in a defined scenario. They are not participation trophies and must not imply professional certification unless an authorized partner provides that certification.

## Required Badge Fields

Every badge must include:

- Name
- Description
- Skill category
- Issuing scenario
- Criteria
- Passing threshold
- Date earned
- Version
- Visibility setting

## Evidence Rules

A badge may be awarded only when:

1. The user completes the required scenario.
2. The scoring requirements are met.
3. No automatic-fail behavior is selected.
4. The result is stored with the scenario version used.

## Badge Levels

The MVP uses one foundational badge. Later badge families may use:

- Foundations
- Proficient
- Advanced

Levels must reflect increasingly difficult evidence, not repeated completion of the same task.

## First Badge

### Customer Service Foundations

**Source:** The Difficult Customer

**Skills assessed:**

- Communication
- Professionalism
- Empathy
- Conflict resolution
- Policy awareness

**Award requirements:**

- Complete the scenario.
- Achieve at least 70 percent overall.
- Receive no automatic-fail result.
- Achieve at least a developing score in each assessed skill.

## Automatic-Fail Examples

- Threatening or insulting the customer
- Sharing protected or private information
- Promising an action outside the employee's authority
- Retaliatory or discriminatory conduct
- Knowingly ignoring an immediate safety concern

## Display Rules

The profile badge detail must show what was assessed and when it was earned. Employers must not see hidden personal coaching notes unless the user explicitly chooses to share them.

## Versioning

When scoring, content, or criteria materially change, the badge or scenario version must change. Existing earned badges retain the version under which they were awarded.

## Integrity

The platform should record attempts and outcomes. Retakes may be allowed, but the profile should display the highest valid result rather than hiding evidence rules from the user.
