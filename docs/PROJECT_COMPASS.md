# Project Compass

## Purpose

Project Compass establishes the product identity, visual system, navigation shell, and reusable interface foundation for Fabled Compass.

This project is not a feature sprint. It is the platform foundation.

## Success Criteria

Project Compass is complete when:

- The application has one consistent shell.
- The dark slate and journey-accent design language is applied.
- The three Journey system is in place.
- Profile has become Portfolio.
- Dashboard is the default workspace after login.
- The Communication Hub drawer exists as a permanent system.
- The Professional Compass placeholder exists.
- Reusable UI components exist for cards, buttons, forms, panels, badges, tooltips, and drawers.
- Existing pages no longer rely on beige or bright white legacy styling.

## Journey System

### Career Journey

Accent: Cyan

Purpose: jobs, portfolio, applications, career readiness, employer-facing evidence.

### Community Journey

Accent: Amber

Purpose: friends, groups, mentors, guilds, networking, events.

### Adventure Journey

Accent: Crimson

Purpose: assessments, workplace challenges, AI roleplay, scenario evidence, competencies.

## Phase 1: UI Kit v1

Deliverables:

- Design tokens
- App Shell
- Sidebar
- Top Bar
- Communication Hub drawer
- Journey-aware accent system
- Core UI components
- Dashboard v2 placeholder
- Portfolio rename support

## Phase 2: Page Migration

Existing pages should be moved into the shell and updated to use the UI kit.

Priority:

1. Dashboard
2. Portfolio
3. Adventures
4. Jobs
5. Account / Navigator Center

## Phase 3: Professional Compass Placeholder

The dashboard and portfolio should include a first version of the Professional Compass.

It may begin as a placeholder visualization, but it must establish the product direction.

## Rule

No major new feature work should begin until Project Compass gives the application a unified foundation.
